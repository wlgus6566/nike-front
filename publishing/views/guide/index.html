Nike D&P Platform 구축 ?..

Nike D&P Platform는 Vue 기반으로 작업 된 프로젝트 입니다.

vue 서버 세팅

vue를 사용하기 위해선 npm, node.js가 필요

Node.js와 NPM 설치하기

npm의 경우는 Node Package Manager이기 때문에 node.js를 설치하면 같이 설치됩니다.


$ node -v

설치 완료 후 하단의 명령 프롬포트를 실행하여 version 확인 한다

npm 버전 확인


$ npm -v
**node 버전 확인** 

$ node -v

Nike D&P Platform 구축 프로젝트에서 사용된 버전은 하단과 같다.

  • npm version - 6.11.3
  • Node version - v10.17.0

package.json에 명시된 라이브러리등 설치


$ npm install

서버 실행


$ npm run serve
  • PC localhost:8081
  • MO localhost:8082**

파일의 경로


<!-- 공통 css -->
<link rel="stylesheet" href="/resources/pc/assets/css/ui.ticketbay.pc.css">
<!-- ui script -->
<script src="/resources/pc/assets/js/library/jquery-3.4.1.min.js"></script>

<!-- 이미지 파일 경로 -->
<img src="/resources/pc/assets/images/images.jpg" alt="대체텍스트" >

표준 문법의 사용

  • html5 명세에 맞게 작성하며, W3C validation을 통과해야 한다.
  • 특수기호는 Entity name을 사용하여 entity 코드로 변환한다. Entity 코드는 ISO-8859-1을 기준으로 한다.

주의 해야할 특수기호 Entity name

Character ₩ " & < >   ·
Entity Name &#8361; &quot; &amp; &lt; &gt; &nbsp; &middot;

파일의 인코딩

  • *.html, *.css, *.js의 파일의 저장 방식을 UTF-8로 설정한다.
문자셋(charset) 기본 인코딩
charset=utf-8 utf-8

코드 들여쓰기

  • 코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 들여쓰기 규칙을 준수한다. 마크업의 중첩이 깊어질 때마다 자식 엘리먼트는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 설정한다.

들여쓰기를 하지 않는 경우


<!doctype html>
<html lang="en">
<head>

</head>
<body>
<div id="wrap">

</div>
</body>
</html>
  • html의 자식 엘리먼트인 head, body
  • body의 자식 첫 번째 엘리먼트 div="wrap"

주석

  • 코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 주석사용을 준수한다.
    •  
    
<%-- header// --%>
<header id="header">
    <a href="#none">ticketbay</a>  <%-- logo --%>
</header>
<%-- //header --%>

<%-- container// --%>
<div id="container">

</div>
<%-- //container --%>

<%-- footer// --%>
<footer id="footer">
</footer>
<%-- //footer --%>
    • 주석 기호와 주석 내용 사이에는 반드시 공백 한 칸이 있어야 한다.
    • 시작과 종료주석의 내용은 동일해야 한다.
    • 한줄 주석의 경우 시작, 종료 주석을 표기하지 않는다.
    • 레이아웃 엘리먼트를 제외한 독립된 콘텐츠 영역의 주석 표기는 선택 사항이다.

    HTML 코드 작성 규칙

    문서형(Document Type Definition) 선언

    • 문서형을 선언하지 않으면 브라우저 호환성을확보하지 못함. 문서형을 선언하는 것은 상호 운용성을 보장하기 위함이다.
    
<!doctype html>

    언어 / 문자셋(charset) 지정

    • lang 애트리뷰트는 User Agent가 언어를 올바로 해석할 수 있게 도와주며, 검색과 음성 장치에 활용된다.
    • utf-8은 다국어 지원이 가능하며, euc-kr 보다 표현 가능한 한글(고어, 음절 등)이 많음.
    • 한글은 utf-8에서 3바이트, euc-kr에서 2바이트를 차지하기 때문에 utf-8이 euc-kr에 비하여 DB 저장 용량이나 트래픽이 많을수 있음.
    
<html lang="en">
<meta charset="utf-8">

    CSS 코드 작성 규칙

    • 문서의 첫 줄에 인코딩을 선언하며 인코딩은 HTML과 동일한 인코딩을 지정한다. 작업 정보를 아래와 같은 방법으로 작성한다.
    
@charset "utf-8";

body , html{margin:0; padding:0;}

    모든 속성은 영문 소문자로만 작성한다

    
/* 잘못된 예 */
.contents{
	color:#FFF;
}

/* 올바른 예 */
.contents{
	color:#fff;
}

    세미콜론(;)

    • 마지막 선언된 속성에도 세미콜론(;)을 사용한다.
    
/* 잘못된 예 */
.contents{
	color:#FFF;
	padding-left:0
}

/* 올바른 예 */
.contents{
	color:#fff;
	padding-left:0;
}

    들여쓰기/줄 바꿈

    • CSS 코드 작성시 속성마다 줄 바꿈, 들여쓰기로 가독성을 높인다.
    
/* 잘못된 예 */
.contents{
	position:relative;
	display:block;
	width:10px;
	height:10px;
	margin:0;
	padding:0;
	font-size:12px;
	color:red;
}
.box,
.box2{
	float:left;
	display:block;
}

/* 올바른 예 */
.contents{
	position:relative;
	display:block;
	width:10px;
	height:10px;
}

    속성의 선언 순서

    • content, z-index, position, top, right, bottom, left, display, flex, visibility, float, clear, overflow, box-sizing, width, height, margin, padding, border 의 순서를 지킨다

}

    주석 / 빈줄

    • CSS 주석 처리는 시작 주석을 작성하며 종료 주석은 작성하지 않는다. 종료코드 이후에 오는 요소에 한 칸 띄어 쓰기를 허용한다.
    • 코드를 구분하기 위하여 코드 그룹 간 1줄의 빈 줄을 허용한다. 빈 줄의 1줄을 초과하지 않는다.

    css 재정의

    • margin, padding의 경우 5px단위로만 정의 하여 사용한다.

    font-family 정의 방법

    
body {
	font-family: "Roboto", "Noto Sans KR", "Arial", "Apple SD Gothic Neo",sans-serif;
}
.bebas {
	font-family: 'Bebas Neue', 'Noto Sans KR', sans-serif !important;
	letter-spacing: 0.5px;
}
    • 1순위로 영문(Roboto), 국문(Noto Sans KR)로 정의한다.
    • 강조 영문 폰트는 Bebas Neue로 정의한다.
    • Bebas Neue 사용시 font-family를 재정의 하거나 클래스 .bebas를 사용한다.
